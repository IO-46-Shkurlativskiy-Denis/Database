# Шкурлатівський Денис ІО-46 Лабораторна Робота №1 Організація баз даних
## Збір вимог та розробка схеми ER

---

## Цілі

- Зрозуміти важливість збору та аналізу вимог у проектуванні баз даних.
- Отримати практичний досвід інтерв'ювання зацікавлених сторін та документування функціональних вимог і вимог до даних.
- Навчитись ідентифікувати сутності, атрибути та зв'язки з вимог (знаходження *ядер* або іменників).
- Створити концептуальну ER-діаграму, яка моделює вимоги до даних.
- Підготуватись до наступних лабораторних робіт: реалізація моделі у вигляді таблиць PostgreSQL, написання SQL-запитів (OLTP/OLAP) та застосування нормалізації та міграцій.

---

## Хід роботи

Адаптований приклад з методичних вказівок:  
Відстеження запасів: Товар, Постачальник, Склад, Замовлення, Клієнт.  

У базі даних зберігатимуться відомості про:  
- Товари(Унікальний ID, назва, категорія, тощо)
- Постачальників товарів(ID, назва компанії, контактні дані)
- Склади(ID, назва, адреса, наявний товар)
- Клієнтів(ID, ім'я, контактні дані)
- Замовлення(ID, час, клієнт, замовлені товари, тощо)

## Пояснення зв'язків

Постачальник може постачати будь-яку кількість різних товарів. Для спрощення припустимо що кожен вид товару може постачати лише один постачальник.

Товар та склад пов'язані через проміжну сутність, що зберігає дані для кожного товару на кожному складі. Склад може мати(або не мати) будь яку кількість різних товарів, а товар має бути присутнім хоч на одному складі.

Товар та замовлення пов'язані через проміжну сутність, що зберігає дані для кожного товару в окремому замовленні. В замовленні обов'язково має бути хоч 1 товар.

Клієнт надає свої дані для замовлення. Один клієнт може мати багато замовлень, але кожне замовлення здійснюється лише одним клієнтом.
